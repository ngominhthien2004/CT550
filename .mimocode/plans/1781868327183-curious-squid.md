# Vue Doctor Frontend Scan & Fix Plan

## Context
The frontend is a Vue 3 + Vite project with **130 `.vue` files**, 15 Pinia stores, 33 views, and 19 component subdirectories. The user wants to diagnose and fix any issues using the `vue-doctor` tool.

## Phase 1: Full Diagnostic Scan
Run a full scan to identify all issues:
```bash
cd frontend && npx vue-doctor@latest . --verbose
```
This will produce a score (80–100 Great, 50–79 Needs work, 0–49 Critical) and a list of diagnostics.

## Phase 2: Analyze Results
Review the scan output to categorize issues by severity and type:
- **Reactivity bugs** (destructure-props, reactive-reassign, ref-no-value)
- **Performance issues** (giant-component, large bundle)
- **Architecture issues** (mixed-api-styles, pinia misuse)
- **Nuxt-specific** (skip — this is pure Vue 3, not Nuxt)

## Phase 3: Apply Fixes
Run auto-fix mode:
```bash
cd frontend && npx vue-doctor@latest . --fix
```
Review each diagnostic and apply the suggested fix hint. For issues that can't be auto-fixed, apply manual fixes following the Vue Doctor common fixes table:
- `reactivity-destructure-props` → use `toRefs(props)` or `props.xxx`
- `reactivity-reactive-reassign` → use `Object.assign(state, newData)`
- `reactivity-ref-no-value` → add `.value` in script
- `perf-giant-component` → split into sub-components (<300 lines)
- `pinia-no-store-to-refs` → use `storeToRefs(store)`
- `pinia-direct-state-mutation` → use actions or `$patch()`
- `arch-mixed-api-styles` → migrate to `<script setup>`

## Phase 4: Verify
Re-run the scan to confirm score improvement:
```bash
cd frontend && npx vue-doctor@latest . --verbose
```

## Verification
- Compare before/after scores
- Run `npm run build` to ensure no compilation errors
- Spot-check any manually fixed components for correctness

## Files Likely Affected
- Various `.vue` files under `frontend/src/views/` and `frontend/src/components/`
- Pinia stores under `frontend/src/stores/`
- Entry point `frontend/src/main.js`
